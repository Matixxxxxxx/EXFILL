# PROJECT EXFIL

# Core Game Loop

Version: 0.1


# 1. Purpose

The Core Game Loop defines the main repeated gameplay experience.

Every major system must support this loop.


# 2. Main Loop


## Step 1 — Preparation

The player prepares for a raid.

Actions:

- choosing equipment;
- selecting weapons;
- organizing inventory;
- checking objectives;
- deciding risk level.


Main emotion:

"Am I ready?"


---

## Step 2 — Deployment

The player enters a dangerous location.


The player does not know everything:

- enemy positions;
- available loot;
- upcoming events;
- other players' plans.


Main emotion:

"Anything can happen."


---

## Step 3 — Exploration

The player searches the environment.

Activities:

- finding loot;
- completing objectives;
- discovering locations;
- collecting information.


Main decision:

"How far should I go?"


---

## Step 4 — Conflict

The player encounters:

- other players;
- AI enemies;
- environmental dangers.


The player decides:

- fight;
- escape;
- hide;
- negotiate;
- change plan.


---

## Step 5 — Extraction

The player attempts to leave.


The final question:


"Do I leave now or risk more?"


---

## Step 6 — Progression

After the raid:

Success:

- keep loot;
- improve equipment;
- progress.


Failure:

- lose equipment;
- learn;
- adapt.


---

# 3. Core Formula


Preparation

+

Risk

+

Decision

+

Consequence

=

EXFIL Experience


# 4. Design Goal

Every raid should create uncertainty.

The player should never feel:

"I already know exactly what will happen."

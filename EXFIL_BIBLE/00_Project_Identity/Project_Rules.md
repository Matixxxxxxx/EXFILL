# PROJECT EXFIL

# Project Rules

Version: 0.1


# 1. General Rules

EXFIL is built as a long-term project.

All decisions must protect:

- gameplay quality;
- technical stability;
- player experience.


# 2. Feature Rules

Before adding a feature:

Check:

1. Why does this exist?
2. What problem does it solve?
3. Does it create interesting gameplay?
4. Can it be maintained?


# 3. Forbidden Design Mistakes


## Feature Quantity Over Quality

Do not add content just to increase numbers.


## Artificial Grinding

Do not create boring repetition only to increase playtime.


## Pay To Win

Players must never be able to buy unfair gameplay advantages.


## Copying Without Purpose

Inspired mechanics must be redesigned for EXFIL.


# 4. Development Rules

Every implemented system must have:

- documentation;
- design explanation;
- technical plan.


Code should never be created before understanding:

- why the system exists;
- how it interacts with others.


# 5. Balance Rules

Strong advantages must have disadvantages.


Examples:

Expensive equipment:

+
better performance

-
higher risk


Powerful weapons:

+
strength

-
cost or difficulty


# 6. Roblox Rules

EXFIL must respect Roblox limitations.

Important priorities:

- performance;
- server stability;
- optimization;
- accessibility.


# 7. AI Development Rules

Any AI assistant working on EXFIL must:

- read documentation first;
- follow architecture;
- avoid creating random systems;
- explain major decisions;
- keep code consistent.


# 8. Golden Rule

Never sacrifice the identity of EXFIL for a quick solution.

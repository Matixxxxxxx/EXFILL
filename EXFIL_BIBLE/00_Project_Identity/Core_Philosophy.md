# PROJECT EXFIL

# Core Philosophy

Version: 0.1

Status: Foundation Phase


# 1. Purpose

This document defines the fundamental philosophy of EXFIL.

Every future system, mechanic and feature must follow these principles.

If a feature conflicts with the core philosophy, it must be redesigned or removed.


# 2. The First Principle

# Every Raid Is A Story

The main value of EXFIL is not the loot.

The main value is the experience created during the raid.

A successful raid is not only:

- extracting expensive items;
- earning currency;
- increasing progression.


A successful raid is when the player experiences:

- tension;
- difficult choices;
- unexpected situations;
- teamwork;
- memorable moments.


Example:

A player escaping with low health and valuable loot is often more memorable than a player safely collecting common items.


# 3. Decisions Over Mechanics

EXFIL is built around player decisions.

A mechanic exists only if it creates meaningful choices.


Examples:

Good:

"Should I continue looting or extract now?"

"Should I fight this team or avoid them?"

"Should I spend expensive medicine or save it?"


Bad:

"Press this button because the game requires it."


Every major system should create situations where the player must think.


# 4. Depth Over Quantity

EXFIL values quality over amount.


We prefer:

- 5 excellent maps instead of 20 empty maps;
- 50 unique weapons instead of 300 identical weapons;
- meaningful quests instead of hundreds of repetitive tasks.


Content must have purpose.


# 5. Realism With Purpose

EXFIL is not a military simulator.

Realism is used only when it improves gameplay.


Good realism:

- weapon handling;
- armor choices;
- limited resources;
- tactical movement.


Bad realism:

- unnecessary complexity;
- boring waiting;
- mechanics that only slow the player.


The question is not:

"Is this realistic?"


The question is:

"Does this make the game better?"


# 6. Risk Creates Value

Without risk, rewards lose meaning.


EXFIL must create situations where players think:


"I have something valuable. Should I leave or risk more?"


High rewards should require:

- danger;
- preparation;
- skill;
- decision making.


# 7. Failure Must Have Meaning

Death is an important part of extraction shooters.

However, failure should never feel pointless.


After losing, players should understand:

- what happened;
- why they failed;
- what they can improve.


The goal:

"I learned something."

Not:

"The game wasted my time."


# 8. Respect Player Time

EXFIL should respect players.

The game should avoid:

- artificial waiting;
- unnecessary grinding;
- repetitive tasks without purpose.


Players should continue playing because they want to.


# 9. Teamwork Creates Stories

Teamplay is one of the strongest sources of memorable moments.


The game should encourage:

- communication;
- cooperation;
- helping teammates;
- tactical decisions.


Examples:

- saving an injured teammate;
- sharing resources;
- planning an extraction.


# 10. The Golden Rule

Before adding anything to EXFIL:

Ask:

"Will players remember this moment?"


If the answer is no, reconsider the feature.

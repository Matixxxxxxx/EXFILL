# PROJECT EXFIL

# Design Principles

Version: 0.1

Status: Foundation Phase


# 1. Purpose

This document defines the rules used when designing every EXFIL system.

All future gameplay decisions should follow these principles.


# 2. Player Experience First

Every system must start with the player experience.

Before creating a feature, answer:

- What should the player feel?
- What decision should the player make?
- What memorable situation can this create?


Example:

Bad approach:

"Add a hunger system because survival games have it."


Good approach:

"Add a resource management system that creates tension during long raids."


# 3. Every System Needs A Reason

No system exists only because it is realistic or interesting.

Every feature must have:

## Purpose

Why does it exist?


## Player Impact

How does it change gameplay?


## Emotional Impact

What feeling does it create?


If a system cannot answer these questions, it should not be added.


# 4. Decisions Create Depth

Depth comes from choices.

Examples:

Equipment:

Choice:
Take expensive armor and risk losing it.

Map:

Choice:
Enter dangerous area for better rewards.

Combat:

Choice:
Fight or avoid.


The goal is not complexity.

The goal is meaningful decisions.


# 5. Avoid Empty Content

Content without purpose damages the game.

Avoid:

- useless items;
- meaningless upgrades;
- repetitive quests;
- weapons with no identity.


Every object should have a role.


# 6. Systems Must Work Together

EXFIL systems should support each other.


Example:

Weapon system:

creates combat choices.


Economy:

creates equipment value.


Loot:

creates risk decisions.


Maps:

create tactical situations.


No system should exist separately.


# 7. Reward Skill And Knowledge

Progression should not only reward time played.

Players should improve through:

- map knowledge;
- aiming;
- tactics;
- decision making.


A new player with knowledge should have a chance against a richer player.


# 8. Build For Long Term

Every system should consider:

- first hour;
- first week;
- hundreds of hours.


A mechanic that is fun for one day but boring later is not successful.


# 9. Simplicity Before Complexity

Start with the simplest version.

Expand only when necessary.

A simple working system is better than a complex unfinished system.


# 10. Final Design Rule

Every feature must answer:

"Does this make EXFIL more memorable?"
